# Monero (XMR) 채굴 시작하기

> 제로부터 작동하는 채굴기 + 대시보드까지 완전한 가이드.
> 목표: 데스크톱 CPU에서 (거의) 돈이 안 되지만, 잘 돌아가는 채굴기
> + 예쁜 TUI 모니터링 화면을 만드는 것.

| 컴퓨터        | 어떤 현대 x86 또는 ARM CPU든 작동. 이 가이드는 macOS 기준. |

| 약 30분       | 대부분 로그 파일 채워질 때까지 대기                       |

## 1단계 — Monero 지갑 주소 받기

두 가지 경로: **거래소 지갑** (빠름, 위탁) 또는 **본인 지갑** (느림, 자기主权). 얼마나 자신을 신뢰하는지 vs 거래소를 신뢰하는지에 따라 선택.

### 경로 A — 거래소 지갑 (5분)

Monero (XMR)를 거래하는 모든 주요 거래소가 입금 주소를 줄 수 있어요. 작성 시점 기준으로 알파벳 순서대로: Binance, Bybit, Kraken, KuCoin, MEXC, OKX 등이 있습니다. **이 대시보드는 그 어떤 거래소와도 호환됩니다** — 자금을 어디에 보관하는지 상관없이 동작해요.

> **거래소 선택 팁:**
> - 해당 국가에서 **XMR 입금이 활성**인 거래소를 선택하세요.
>   일부 거래소(특히 미국/한국 사용자를 위한)는 XMR 입금을 주기적으로
>   비활성화합니다.
> - 첫 번째 거래소에 XMR이 없으면 다른 거래소를 시도하세요 —
>   위 목록은 모두 나열된 게 아니에요.

#### 일반적인 흐름 (모든 거래소 공통)

```
  +--------------------------------------------------------------+
  |  1.  회원가입 + KYC 완료 (여권, 셀카, 주소)                  |
  |  2.  2단계 인증 (2FA) 켜기 (Google Authenticator, 백업 코드)  |
  |  3.  이동:  지갑  ->  입금                                   |
  |  4.  검색:  XMR  또는  Monero                                |
  |  5.  선택:  "Monero (XMR)"   (wrapped XMR 절대 아님)         |
  |  6.  네트워크:  XMR  (ERC20, BEP20 등 절대 금지)             |
  |  7.  주소 복사 — 4 또는 8로 시작                              |
  |  8.  길이 95~106자 확인                                      |
  +--------------------------------------------------------------+
```

#### 가짜 XMR 토큰 구분하기 (매우 중요)

거래소에서 "XMR"을 검색하면 **여러 항목**이 나옵니다 — 대부분 **다른 체인의 스캠 토큰**이에요. 진짜를 고르는 법:

| 신호                  | 진짜 XMR                              | 가짜 XMR (스캠)                       |
| --------------------- | ------------------------------------- | ------------------------------------- |
| **이름**              | `Monero (XMR)`                        | `XMR`, `Wrapped Monero`, `MONERO`     |
| **네트워크**          | `Monero` 또는 `XMR`                   | `Solana`, `Ethereum`, `BSC`, `Base`   |
| **가격**              | ~$300 (CoinGecko와 일치)               | $0.00001 ~ $0.10                      |
| **주소 형식**         | 95~106자, `4` 또는 `8`로 시작          | 32~44자, `0x` 또는 `6`으로 시작        |
| **유동성**            | 수억 USD                               | $0 ~ $50K                             |
| **일 거래량**         | $100M+                                | $0 ~ $5K                              |

> **가장 확실한 단일 체크:** 복사한 주소가 `4` 또는 `8`로 시작하고
> 95자 이상이어야 해요. 만약 그렇지 않다면, **가짜를 복사한 것**이에요.

### 경로 B — 본인 지갑 (10분, 장기적으로 추천)

진정한 소유권을 원한다면 공식 Monero 지갑을 설치하세요. 거래소가
자금을 가지고 있으면 사라질 수 있지만, 공식 지갑은 당신 거예요.

```
  +--------------------------------------------------------------+
  |  1.  https://www.getmonero.org/downloads/ 방문               |
  |  2.  OS에 맞는 "Monero GUI" 다운로드                         |
  |  3.  설치 후 실행                                            |
  |  4.  선택:  "Create a new wallet"                            |
  |  5.  25단어 시드 니모닉을 **종이에** 적기                     |
  |      (텍스트 파일 X, 스크린샷 X)                             |
  |  6.  지갑 비밀번호 설정                                       |
  |  7.  데몬이 블록체인 동기화할 때까지 대기                     |
  |  8.  이동:  Receive  ->  기본 주소 복사                       |
  +--------------------------------------------------------------+
```

```
  ┌────────────────────────────────────────────────────────────┐
  │  경고: 25단어 시드는 지갑을 복구할 수 있는 유일한 방법.   │
  │  분실 시 XMR는 영원히 사라져요. 타인에게 노출 시 XMR는     │
  │  영원히 사라져요. 종이에 적어서 물리적으로 안전한 곳에     │
  │  보관. 사진 찍지 마세요. 웹사이트/채팅/클라우드에 입력     │
  │  금지.                                                    │
  └────────────────────────────────────────────────────────────┘
```

### 어떤 경로를 선택해야 할까?

| 상황                                  | 경로          |
| ------------------------------------- | ------------- |
| 오늘 오후에 채굴을 시도해보고 싶음    | A (거래소)    |
| 몇 달 동안 채굴할 계획                | B (본인 지갑) |
| $50 이상 가치의 XMR을 채굴할 계획     | B (본인 지갑) |
| 그냥 호기심 / 학습                    | A (거래소)    |

나중에 A → B로 이동 가능: 거래소 주소로 채굴 → 본인 지갑으로 출금.

## 2단계 — XMRig 설정

### XMRig 설치

**macOS (Homebrew):**
```bash
brew install xmrig
```

**다른 플랫폼:** <https://xmrig.com/download> 참조

### 설정 파일 생성

`~/.xmrig/xmrig-config.json`에 파일 생성 (또는 다른 위치):

```bash
mkdir -p ~/.xmrig
nano ~/.xmrig/xmrig-config.json
```

이 템플릿 붙여넣고, **지갑 주소와 워커명 교체**:

```json
{
  "autosave": true,
  "cpu": true,
  "opencl": false,
  "cuda": false,
  "pools": [
    {
      "url": "pool.supportxmr.com:5555",
      "user": "PASTE_YOUR_XMR_ADDRESS_HERE",
      "pass": "my-mac-mini",
      "keepalive": true,
      "tls": false,
      "nicehash": false
    }
  ]
}
```

#### 필드 설명

| 필드        | 의미                                                      |
| ----------- | --------------------------------------------------------- |
| `url`       | 채굴 풀 주소 (host:port)                                  |
| `user`      | 지갑 주소 (95자 이상, `4` 또는 `8`로 시작)                |
| `pass`      | 워커명 — 이 머신을 식별할 수 있는 아무 이름              |
| `keepalive` | 풀 연결 끊기면 자동 재연결                                |
| `tls`       | 풀과 암호화 연결 (느리지만 더 사적)                       |
| `nicehash`  | NiceHash로 채굴 시 `true`, 아니면 `false`                 |

### 채굴 풀 선택

위 기본값은 **SupportXMR**을 가리키는데, 잘 알려진 안정적인 Monero 풀이에요. 다른 인기 옵션:

| 풀            | URL                              | 수수료 | 비고                                      |
| ------------- | -------------------------------- | ------ | ----------------------------------------- |
| **SupportXMR** | `pool.supportxmr.com:5555`     | 0.6%   | 기본값. 크고 안정적, 초보자 친화적        |
| MoneroOcean   | `gulf.moneroocean.stream:10128` | 0%     | 알고리즘 자동 전환                        |
| P2Pool        | `p2pool.io:3333`                 | 0%     | 완전 탈중앙화, 풀 운영자 없음             |
| HashVault     | `pool.hashvault.pro:3333`        | 0.9%   | 솔로 채굴에 좋음                          |
| ViaBTC        | `viabtc.com:3333`                | 2%     | 대형, 멀티코인                            |

> 이 대시보드는 **어떤 풀과도 동작**합니다 — 단순히 로그 파일을 읽을 뿐.
> URL이 가장 편한 풀을 선택하세요.

## 3단계 — 실행 및 모니터링

### 채굴기 시작

```bash
xmrig --config=~/.xmrig/xmrig-config.json
```

이런 출력이 보일 거예요:

```
 * ABOUT        XMRig 6.26.0
 * CPU          Intel(R) Core(TM) i5-8500B
 * POOL #1      pool.supportxmr.com:5555
 * READY threads 5/5
```

> **주의:** 채굴은 CPU를 많이 사용해요. 머신이 따뜻해지고 팬이 빨라져요.
> 정상입니다. 다른 작업을 위해 머신을 쾌적하게 유지하고 싶다면
> 메인 README의 **일시정지/재개** 섹션을 참조하세요.

### 대시보드 시작

별도 터미널에서:

```bash
git clone https://github.com/sigco3111/xmrig-dashboard
cd xmrig-dashboard
python3 -m pip install --user textual psutil
python3 miner-dashboard.py
```

대시보드는 기본적으로 `~/.xmrig/xmrig.log`와 `~/.xmrig/xmrig-config.json`을
자동으로 찾습니다. 다른 위치에 있다면:

```bash
XMRIG_LOG=/path/to/xmrig.log XMRIG_CONFIG=/path/to/config.json \
  python3 miner-dashboard.py
```

### 무엇을 봐야 하나

```
  +-----------------------------------------------------------+
  |  XMRig 콘솔에서 "READY" 기다리기 (10-15초)                |
  |                                                           |
  |  "accepted" 라인 기다리기 (보통 1-2분마다 한 번)           |
  |     -> 풀이 share를 수락했다는 뜻                         |
  |     -> "share"는 작업 증명의 한 조각                      |
  |     -> 많은 share = 블록 보상의 작은 일부                  |
  |                                                           |
  |  대시보드 HASHRATE 패널 관찰                               |
  |     -> 1.0-1.5 KH/s (현대 데스크톱 CPU) = 정상            |
  |     -> "max"는 이 세션에서 달성한 최고치                   |
  |                                                           |
  |  Earnings 패널은 정보 제공용                               |
  |     -> 1 KH/s에서 ~$0.04/일                               |
  |     -> 커피 한 잔... 3개월치                               |
  +-----------------------------------------------------------+
```

## 자주 묻는 질문

**Q: 30초 지났는데 hashrate가 0 H/s.**
A: XMRig이 로그 파일에 아직 안 쓴 거예요. 다른 터미널에서
`tail -f ~/.xmrig/xmrig.log` 실행 — "READY"와 "accepted" 라인이 보여야 해요.

**Q: "Connection refused" 에러.**
A: 네트워크나 방화벽이 풀을 막고 있어요. 위 표의 다른 풀 URL을 시도하세요.

**Q: 채굴 때문에 배터리가 죽거나 노트북이 뜨거워요.**
A: 대시보드에서 `p` 키로 일시정지, 또는
`pkill -STOP -f "xmrig --config"`로 SIGSTOP 전송. `r` 또는
`pkill -CONT`로 재개.

**Q: 여러 주소로 동시에 채굴할 수 있나요?**
A: 가능하지만 이 대시보드로는 안 됩니다. XMRig 설정에 여러 풀을
추가하세요. 대시보드는 첫 번째 풀만 표시합니다.

**Q: 내 안티바이러스가 XMRig을 차단해요.**
A: 일부 AV가 채굴기를 "PUA" (잠재적 원치 않는 앱)로 표시해요.
<https://xmrig.com>에서 받은 정식 XMRig 바이너리에 대한 알려진
오탐입니다. Homebrew로 설치했다면 안전해요. 다른 출처라면
SHA-256 체크섬을 확인하세요.
