## 지갑 시드 (본인 지갑만 해당)

공식 Monero GUI 지갑을 사용했고 25단어 시드를 받았다면:

```
  해야 할 것:
  - 시드를 종이에 펜으로 적기
  - 종이를 물리적으로 안전한 곳에 보관
  - 한 부 만들어서 다른 물리적 위치에 보관
  - 백업이 읽을 수 있는지 확인 (번지지 않음, 퇴색 안 됨)

  하지 말아야 할 것:
  - 시드를 사진 찍기 (휴대폰 백업이 클라우드에 동기화됨)
  - 어떤 웹사이트, 채팅, 문서에 시드 입력
  - 클라우드 동기화되는 비밀번호 관리자에 시드 저장
  - 이메일로 자기 자신에게 보내기
  - 네트워크 프린터로 출력 (프린터 로그는 사적이지 않음)
```

> **시드 = 지갑.** 시드를 가진 사람이 XMR을 소유해요. 금고
> 열쇠처럼 다루세요.

## 지갑 주소 프라이버시

Monero 주소가 **공개되면 영원히 공개**돼요 — Monero 블록체인은
주소를 영구적으로 기록하고, 블록체인 분석 회사들이 시간이 지나면서
다른 데이터와 연관시켜요.

### 주소가 "공개"된다는 것의 의미

| 공개된 위치                              | 결과                                   |
| ---------------------------------------- | -------------------------------------- |
| 본인 xmrig-config.json                   | 안전 — 본인 파일                       |
| 공개 GitHub repo                         | **영구 공개 기록**                     |
| 공유 문서, 채팅, 스크린샷                | 아마 영구적, 하지만 내릴 수 있음        |
| SupportXMR 웹 대시보드                   | **공개** — 주소 가진 누구나 hashrate 볼 수 있음 |
| 코인조인, 거래소, KYC 필요 서비스        | 신원까지 추적됨                        |

> 주소는 **시드와 같지 않아요.** 주소를 공유하면 사람들이 XMR을
> 보낼 수 있어요(또는 잔액을 볼 수 있어요). XMR을 쓸 수 있는 권한은
> 안 줘요. 하지만 주소가 KYC 서비스에 닿으면 의사 익명성이 깨져요.

### 이 repo가 어떻게 처리하나

```
  +--------------------------------------------------------------+
  |  miner-dashboard.py:                                         |
  |    - 하드코딩된 지갑 주소 없음                                |
  |    - 하드코딩된 워커명 없음                                   |
  |    - 하드코딩된 거래소명 없음                                 |
  |    - 지갑/워커는 런타임에 xmrig-config.json에서 로드           |
  |                                                              |
  |  .gitignore:                                                 |
  |    - xmrig-config.json 기본 무시                              |
  |    - *.log 무시 (로그도 주소 누설 가능)                       |
  |                                                              |
  |  README:                                                     |
  |    - 예시는 "PASTE_YOUR_XMR_ADDRESS" placeholder              |
  |    - 실제 주소를 git에 커밋하지 마세요                         |
  +--------------------------------------------------------------+
```

이 repo를 fork해서 실수로 실제 `xmrig-config.json`을 커밋했다면,
**주소가 영구 누설**돼요. 정리하려면:

1. `git filter-branch` 또는 `git filter-repo`로 히스토리에서 파일 제거
2. 정리된 히스토리 force-push
3. **주소는 여전히 누설됨** — 공개된 것으로 가정, 중요하다면 새 주소로 이동

## 거래소 계정 보안

거래소 입금 주소를 사용했다면:

### 1. 2단계 인증 (2FA)

```
  우선순위 순서대로 2FA 활성화:
    1. 하드웨어 보안 키 (YubiKey, Titan)  - 가장 강함
    2. TOTP (Google Authenticator, Authy)  - 좋음
    3. SMS                                    - 약함 (SIM 스왑 공격)

  2FA 백업 코드를 비밀번호 관리자에 저장.
  자금을 입금하기 전에 새 브라우저에서 로그인 테스트.
```

### 2. 출금 주소 화이트리스트

대부분의 거래소가 출금 주소 화이트리스트를 지원해요. 활성화하면,
XMR을 사전 승인한 주소로만 출금 가능. 비밀번호를 얻은 공격자가
계정을 비우는 것을 막아요.

### 3. 피싱 방지

```
  +----------------------------------------------------------+
  |  항상 북마크나 직접 입력한 URL로 거래소에 접속.           |
  |  거래소로부터의 이메일에 있는 링크는 클릭하지 마세요.     |
  |                                                           |
  |  브라우저에 실제 거래소 URL을 북마크.                      |
  |  거래소마다 고유한 비밀번호 사용.                          |
  |  거래소의 피싱 방지 코드 설정 (제공하는 경우).            |
  +----------------------------------------------------------+
```

## 채굴 머신 보안

### 전용 사용자로 XMRig 실행 (Linux)

```
  # 낮은 권한 사용자 생성
  sudo useradd -m -s /bin/bash miner

  # 사용자로 전환
  sudo -u miner -i

  # 이 사용자 홈에서 XMRig 실행
  ~/xmrig --config=~/xmrig-config.json
```

이렇게 하면 XMRig에 보안 취약점이 생겨도 공격자는 낮은 권한
사용자에 접근 — 본인의 일반 계정이 아님.

macOS에서는 더 어려워요. LaunchAgent 예제가 본인의 사용자로
실행되는데, 개인 머신에서는 괜찮지만 서버에는 부적절.

### XMRig 바이너리 검증

XMRig 웹사이트가 모든 릴리스의 SHA-256 체크섬을 게시해요. 실행 전 확인:

```
  # 공식 릴리스 다운로드
  curl -L -o xmrig-6.26.0.tar.gz https://github.com/xmrig/xmrig/releases/download/v6.26.0/xmrig-6.26.0.tar.gz

  # 공식 SHA-256 받기
  curl -L https://github.com/xmrig/xmrig/releases/download/v6.26.0/SHA256SUMS | head -3

  # 검증
  shasum -a 256 xmrig-6.26.0.tar.gz
```

두 해시가 일치해야 해요. 일치하지 않으면 바이너리를 실행하지 마세요.

> Homebrew로 설치했다면 (`brew install xmrig`), Homebrew가 자동으로
> 체크섬을 검증해요. macOS 사용자에게 권장하는 경로.

### 네트워크 노출

XMRig은 기본적으로 풀에 **아웃바운드 연결만** 만들어요. 인바운드
연결을 대기하지 않아요. 대시보드도 아웃바운드 요청만 만들어요
(CoinGecko API). 둘 다 머신에서 포트를 열지 않아요.

> 확인하려면 XMRig이 실행 중일 때 `netstat -an` 실행. 풀 IP에
> 하나의 확립된 아웃바운드 연결만 보여야 하고, XMRig 관련 다른 것은
> 없어야 해요.

### 크립토재킹 감시

채굴기 시작하지 않았는데 hashrate가 급증하거나, CPU가 명백한 이유 없이
계속 100%라면 크립토재커가 있을 수 있어요. 일반적인 경로:

- 손상된 브라우저 확장
- 숨겨진 채굴기가 있는 불법 소프트웨어
- 웹사이트에서 JavaScript 채굴기 실행 (요즘은 드물음)

방어:
- 브라우저 확장 감사 (안 쓰는 것 제거)
- Activity Monitor → "Energy" 탭 확인 → 높은 CPU 프로세스 종료
- `htop` 또는 Activity Monitor → 프로세스 이름에 `xmrig`, `minerd`, `cpuminer` 찾기

## 전기 요금 보호

24시간 데스크톱 채굴은 대략 1 kWh 사용해요. 일반 $0.15/kWh
요금제에서, 하루 **$0.04** 전기요금. 하지만 버그로 폭주할 수 있어요:

- XMRig에 `--max-threads` 옵션 있음 — CPU 사용량 제한
- 대시보드의 `p` 키가 즉시 채굴 일시정지
- macOS 사용자 → Energy Saver 설정이 폭주 방지 가능

하드 리밋을 원한다면, XMRig 설정 편집:

```json
{
  "cpu": {
    "enabled": true,
    "max-threads-hint": 50
  }
}
```

이건 XMRig을 1코어의 50%로 제한. 더 낮은 숫자 = 더 낮은 전력 =
더 낮은 hashrate = 더 낮은 수익. 원하는 트레이드오프 선택.

## 이 대시보드가 하지 않는 것

완전한 투명성을 위해:

| 동작                                          | 이 대시보드? |
| --------------------------------------------- | ------------ |
| 지갑 주소 어딘가에 전송                        | 아니오       |
| hashrate를 제3자에게 전송                      | 아니오       |
| 머신에서 네트워크 포트 열기                    | 아니오       |
| 파일 쓰기 (표준 설정 외)                       | 아니오       |
| XMRig 설정 파일 수정                           | 아니오       |
| 종료 후 백그라운드 프로세스 실행               | 아니오       |

유일한 아웃바운드 네트워크 호출은 `GET api.coingecko.com`으로,
XMR/USD 가격을 위해 5분마다, 응답 캐시. Little Snitch나 유사한
네트워크 모니터로 대시보드 실행해서 확인 가능.
