# SupportXMR (및 유사 Monero 풀) 사용법

> 이 가이드는 메인 README에서 권장하는 기본 풀인 SupportXMR에
> 초점을 맞춥니다. 대시보드 자체는 **어떤 풀과도 동작**해요 —
> 풀마다 다른 건 XMRig 설정의 URL과 통계를 확인할 웹 대시보드뿐이에요.

---

## 채굴 풀이란?

데스크톱 CPU에서 Monero 블록을 단독으로 찾으려면 하루에 수백만 분의 1
확률이에요. **채굴 풀**은 많은 채굴자의 작업을 합쳐서 더 빠르게 블록을
찾고, 기여도에 따라 보상을 나눠요.

```
  단독 채굴                  풀 채굴
  +-----------+              +-----------+
  |   나      |              |   나      |---+
  |  1 KH/s   |              |  1 KH/s   |   |
  +-----------+              +-----------+   |  많은
        |                          |         |  채굴자가
        v                          v         v  합쳐짐
  ~1년에 블록 1개             +-------------------+
  (~ $300 전부)               |       풀          |
                             |  (작업 결합)      |
                             +-------------------+
                                      |
                                      v
                                ~2분에 블록 1개
                                (수천 명에게
                                 기여도에 따라
                                 ~$300 분배)
```

블록당 적게 벌지만 꾸준히 벌어요. CPU 1 KH/s에서는 며칠에 한 번
0.0001 XMR 정도 벌 수 있어요. 커피 한 잔 값어치 (1년치). 핵심은
돈이 아니라 모니터링이에요.

## SupportXMR 설정

### 1. 계정 생성 (선택이지만 권장)

계정 없이도 채굴 가능 — 풀이 지갑 주소로 추적해요. **계정이 있으면**
웹 대시보드, 이메일 알림, 설정 가능한 페이아웃 임계값을 받아요.

```
  +--------------------------------------------------------+
  |  1.  https://supportxmr.com/ 방문                    |
  |  2.  "Sign Up" 클릭 (우상단)                           |
  |  3.  이메일 + 강한 비밀번호 입력                        |
  |  4.  이메일 인증                                       |
  |  5.  로그인                                            |
  +--------------------------------------------------------+
```

### 2. 대시보드에 지갑 주소 추가

로그인 후, 하나 이상의 지갑 주소를 등록할 수 있어요. 이렇게 하면
통계를 볼 때마다 주소를 입력하지 않아도 풀에서 주소별 통계를 보여줘요.

```
  +--------------------------------------------------------+
  |  1.  로그인                                            |
  |  2.  이동:  My Account  ->  Wallet Settings            |
  |  3.  "Add wallet" 클릭                                 |
  |  4.  XMR 주소 붙여넣기 (xmrig-config.json의 "user" 필드)|
  |  5.  저장                                              |
  +--------------------------------------------------------+
```

> 여러 주소를 추가할 수 있지만, 대시보드의 "통계 조회" 흐름에는
> 첫 번째만 사용돼요.

### 3. XMRig 설정

메인 README를 따랐다면 이미 완료. 설정의 풀 URL 확인:

```json
{
  "pools": [
    {
      "url": "pool.supportxmr.com:5555",
      "user": "여기에-본인-XMR-주소",
      "pass": "아무-워커명"
    }
  ]
}
```

`pass` 필드는 워커 라벨일 뿐이에요. 여러 머신이 있으면 식별 가능한
이름 사용: `mac-mini-living-room`, `desktop-home` 등.

## SupportXMR 웹 대시보드 읽기

### 메인 통계 페이지

채굴기 실행 후:

```
  https://supportxmr.com/
```

검색/조회 박스에 지갑 주소 붙여넣기. 이런 페이지가 보여요 (요약):

```
  +----------------------------------------------------------------+
  |  Hashrate (last 1h)         |    1,189 H/s                    |
  |  Hashrate (last 24h)        |    1,173 H/s                    |
  |  Hashrate (last 7d)         |    1,180 H/s                    |
  +----------------------------------------------------------------+
  |  Pending balance            |    0.000085 XMR  ($0.027)       |
  |  Confirmed balance          |    0.000012 XMR  ($0.004)       |
  |  Total paid                 |    0.000000 XMR  ($0.000)       |
  +----------------------------------------------------------------+
  |  Shares (valid / invalid)   |    142 / 0  (100% accepted)     |
  |  Last share                 |    2026-06-24 09:18:00 UTC      |
  +----------------------------------------------------------------+
```

#### 각 지표의 의미

| 지표                    | 알려주는 것                                           |
| ----------------------- | ----------------------------------------------------- |
| **Hashrate (1h/24h/7d)** | 채굴기가 하는 일의 양, 평균                            |
| **Pending balance**      | 아직 충분히 블록이 확인 안 되어 지급되지 않은 수익    |
| **Confirmed balance**    | 지급 준비 완료된 수익                                 |
| **Total paid**           | 지갑으로의 평생 지급액                                |
| **Shares (valid/invalid)**| 네트워크 지연 vs 풀이 기대한 솔루션 시간            |
| **Last share**           | 풀이 마지막으로 채굴기에서 들은 시각                   |

> TUI 대시보드의 `accepted` 수가 여기서 "valid shares" 수와 일치해요.
> 두 곳에서 같은 숫자를 보고하는 거예요.

### 워커 내역

특정 워커를 클릭하거나 (또는 아래로 스크롤하면) 워커별 내역이 보여요:

```
  +----------------------------------------------------------------+
  |  Worker               |  Hashrate  |  Shares  |  Last share   |
  |  -------------------- | ---------- | -------- | ------------- |
  |  mac-mini-living-room |  1,189 H/s |    142   |  2 min ago    |
  +----------------------------------------------------------------+
```

> 여러 워커(예: 여러 머신)가 있으면 SupportXMR 페이지가 모든 워커를
> 한 표에 보여줘요.

## 페이아웃

### 언제 지급되나요?

**Confirmed balance**가 **최소 페이아웃 임계값**에 도달하면 풀이
자동으로 지급해요. 기본값은 **0.004 XMR** (현재 가격으로 ~$1.30).
데스크톱 CPU에서는 이 금액에 도달하는 데 몇 주~몇 달 걸려요.
대시보드의 TUI 추정이 실시간으로 보여줘요.

### 페이아웃 임계값을 변경하려면?

```
  +--------------------------------------------------------+
  |  1.  supportxmr.com에 로그인                           |
  |  2.  이동:  My Account  ->  Wallet Settings            |
  |  3.  "Minimum payout" 필드 조정                         |
  |  4.  저장                                              |
  +--------------------------------------------------------+
```

더 빠른 (그러나 네트워크 수수료가 더 많은) 지급을 위해 0.001 XMR
(~$0.32)까지 낮출 수 있어요. 또는 더 적은 지급과 낮은 수수료를 위해
더 높게 설정할 수도 있어요.

### 지급은 어디로 가나요?

풀이 XMRig에서 사용한 지갑 주소로 지급해요. **거래소** 입금 주소를
사용했다면 XMR이 거래소 계정에 들어가요. **본인** 주소를 사용했다면
로컬 지갑에 들어가요.

> **중요:** 거래소 주소를 사용했다면, 그 거래소가 여전히 XMR
> 입금을 지원하는지 확인하세요. 일부 거래소는 특정 지역의 신규
> 사용자에 대해 XMR을 비활성화했어요. 그 경우 지급이 실패하고
> XMR이 풀에 무기한 남아있어요.

## 풀이 다운되면?

SupportXMR가 다운되면, XMRig이 자동으로 재시도해요 (설정의
`keepalive: true` 덕분). 로컬 채굴은 **계속 동작**해요 — 풀이
다시 돌아올 때 share가 수락되지 않을 뿐. 풀이 돌아오면 hashrate가
다시 카운트돼요.

SupportXMR가 살아있는지 확인:

```
  https://supportxmr.com/   ->  페이지 로드되면 풀 작동 중
```

백업 풀로 페일오버하고 싶다면, XMRig 설정에 두 번째 풀 추가:

```json
{
  "pools": [
    {
      "url": "pool.supportxmr.com:5555",
      "user": "본인-XMR-주소",
      "pass": "worker-1",
      "keepalive": true
    },
    {
      "url": "gulf.moneroocean.stream:10128",
      "user": "본인-XMR-주소",
      "pass": "worker-1",
      "keepalive": true
    }
  ]
}
```

XMRig이 순서대로 시도하고 자동 페일오버해요. TUI 대시보드는 현재
활성 풀을 보여줘요.

## SupportXMR 수수료 구조

| 항목              | 수수료                                                  |
| ----------------- | ------------------------------------------------------- |
| 풀 수수료         | 블록 보상의 0.6%                                        |
| 페이아웃 수수료    | 네트워크 트랜잭션 수수료 (풀이 아닌 Monero 네트워크에 지불) |
| 계정 수수료       | 없음                                                    |
| 출금 수수료       | 없음 (자동 지급)                                        |

풀은 비활성, 여러 워커 보유, 웹 대시보드 사용에 대해 수수료를 받지 않아요.

## API 접근 (고급)

SupportXMR는 스크립트에서 쿼리할 수 있는 JSON API를 노출해요. TUI
대시보드는 가격을 위해 CoinGecko를 쓰지만, SupportXMR도 자체 API가 있어요.

```
  GET https://supportxmr.com/api/miner/<본인-지갑-주소>/stats
```

웹 페이지와 같은 모든 숫자가 있는 JSON 문서를 반환해요. 자체 모니터링
스크립트에 유용.

## 다른 풀 (간단히)

| 풀            | 선택할 때                                              |
| ------------- | ------------------------------------------------------ |
| **SupportXMR** | 기본. 초보자 친화적, 큰 hashrate, 안정적.             |
| **MoneroOcean** | 작은 코인 지원 원함; 알고리즘 자동 전환.            |
| **P2Pool**     | 완전한 탈중앙화 원함; 기술적 편안함 필요.             |
| **HashVault**  | 솔로 채굴 느낌; 풀은 블록을 찾으면 전액 보상 지급.   |
| **ViaBTC**     | 이미 다른 코인으로 ViaBTC 사용자.                     |

TUI 대시보드는 모두와 동작해요. 전환하려면 XMRig 설정의 `url`만 바꾸고
XMRig 재시작.
